 Okay, I understand. If you decide to move your OpenVPN server to a different IP address (like your WiFi IP
  for local network testing, or a public IP for wider access), you'll need to regenerate the OpenVPN
  configuration and client files.


  This process will effectively reset your OpenVPN server's configuration and invalidate any previously
  generated client `.ovpn` files. You will need to re-download and re-import the new .ovpn file on any client
   devices (like your Kali VM).


  Here are the commands, step-by-step. Replace `YOUR_NEW_VPN_SERVER_IP` with the actual IP address you want
  to use.

  ---

  1. Stop and Remove the Existing OpenVPN Server Container:

  First, stop and remove the currently running OpenVPN container.



   1 docker stop openvpn_server
   2 docker rm openvpn_server


  ---

  2. Clean Up Existing OpenVPN Data:

  This step is crucial. You need to delete all the old configuration and certificate files.



   1 del /S /Q D:\Think\vulnverse\openvpn-data\*

  (This command is for Windows Command Prompt. If you are using PowerShell, use `Remove-Item -Recurse -Force
   D:\Think\vulnverse\openvpn-data\`)*

  ---

  3. Re-generate OpenVPN Configuration with the New IP:


  Now, generate the base configuration again, but this time with your new IP address.




   1 docker run -v D:/VULNverse/vulnverse/openvpn-data:/etc/openvpn --log-driver=none --rm kylemanna/openvpn ovpn_genconfig -u udp://YOUR_NEW_VPN_SERVER_IP

  Remember to replace `YOUR_NEW_VPN_SERVER_IP` with the actual IP (e.g., `192.168.1.100` for your WiFi IP,
  or your public IP).

  ---

  4. Re-initialize OpenVPN PKI:


  You will be prompted to enter a passphrase for the CA key. Remember this passphrase! You can reuse the old
   one or set a new one. You will also be prompted for the Common Name (e.g., VulnVerse CA).



   1 docker run -v D:/VULNverse/vulnverse/openvpn-data:/etc/openvpn --log-driver=none --rm -it kylemanna/openvpn ovpn_initpki nopass


  ---

  5. Re-generate Client Certificate:


  This will create a new client certificate. You can use client as the name. You will be prompted for the CA
   passphrase you just set.



   1 docker run -v D:/VULNverse/vulnverse/openvpn-data:/etc/openvpn --log-driver=none --rm -it kylemanna/openvpn easyrsa build-client-full client nopass

  (If you want a passphrase for the client key, remove `nopass`.)

  ---

  6. Re-generate the Client `.ovpn` File:

  This will create the new .ovpn file with the updated server IP.



   1 docker run -v D:/VULNverse/vulnverse/openvpn-data:/etc/openvpn --log-driver=none --rm kylemanna/openvpn ovpn_getclient client > D:/VULNverse/vulnverse/openvpn-data/client.ovpn


  ---

  7. Start the OpenVPN Server Container with the New Configuration:



   1 docker run -v D:/VULNverse/vulnverse/openvpn-data:/etc/openvpn -d -p 1194:1194/udp --cap-add=NET_ADMIN --network vulnverse_network --name openvpn_server kylemanna/openvpn


  ---

  After completing all these steps:


   * Download the new `client.ovpn` file from your frontend dashboard.
   * Import this new file into your OpenVPN client on your Kali VM (or any other client device).
   * Connect to the VPN using the new configuration.
   * Test connectivity to your vulnerable machines.


  This comprehensive set of commands will allow you to change the OpenVPN server's IP address.
